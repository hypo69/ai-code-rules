# Регламент организации домашнего сервиса потокового видео

## 1. Назначение

Настоящий регламент определяет архитектуру домашнего сервиса потокового воспроизведения видео через Интернет.

---

# 2. Исходные условия

* Домашний компьютер подключен к роутеру по сети Ethernet (LAN).
* Роутер подключен к Интернету по оптоволоконной линии.
* Скорость исходящего канала составляет около **250 Мбит/с**.
* Используется собственный веб-интерфейс со встроенным HTML5-плеером.
* Публичный домен — **davidka.net**.
* DNS домена обслуживается через **Cloudflare**.
* Максимальное качество воспроизведения — **Full HD (1080p)**.

---

# 3. Архитектура

Сервис работает по следующей схеме:

```text
Пользователь
        │
        ▼
kino.davidka.net
        │
HTTP Redirect
        │
        ▼
URL Cloudflare Tunnel
        │
        ▼
Домашний веб-сервер
        │
        ▼
Встроенный видеоплеер
```

Использование HTTP-редиректа является штатным решением.

Допускается отображение технического URL туннеля в адресной строке браузера пользователя.

Использование обратного прокси (Reverse Proxy) не является обязательным.

---

# 4. Требования к видеоконтенту

Для обеспечения прямого воспроизведения без транскодирования рекомендуется использовать:

* контейнер **MP4**;
* видеокодек **H.264 (AVC)**;
* аудиокодек **AAC**;
* максимальное разрешение **1920×1080**.

Сервер не выполняет транскодирование видео и передает файлы в исходном виде.

---

# 5. Производительность

При отсутствии транскодирования основная нагрузка приходится на сетевой интерфейс.

При исходящей скорости около **250 Мбит/с** возможно обслуживание примерно **30–40** одновременных потоков Full HD со средним битрейтом **5–8 Мбит/с**.

---

# 6. Использование Cloudflare Tunnel

Для публикации домашнего сервиса используется **Cloudflare Tunnel**.

Требования:

* туннель работает постоянно;
* домашний IP-адрес не публикуется;
* соединение осуществляется по HTTPS;
* управление DNS выполняется через Cloudflare.

---

# 7. Доступ пользователей

Пользователям предоставляется короткий публичный адрес:

```text
https://kino.davidka.net
```

При обращении к нему выполняется HTTP-перенаправление на активный URL Cloudflare Tunnel.

Отображение технического адреса туннеля после перенаправления считается допустимым.

---

# 8. Требования к веб-серверу

Веб-сервер должен обеспечивать:

* передачу видеофайлов без изменения содержимого;
* поддержку встроенного HTML5-плеера;
* поддержку HTTP Range Requests для корректной перемотки видео;
* одновременную обработку нескольких подключений.

---

# 9. Ограничения

Архитектура не предусматривает:

* транскодирование видео;
* поддержку разрешений выше Full HD (1080p);
* использование Plex;
* использование Jellyfin;
* использование медиасерверов с дополнительной логикой;
* обязательное применение VPN.

---

# 10. Основной принцип

Архитектура должна оставаться максимально простой.

Домашний компьютер выполняет только хранение и передачу видеофайлов через собственный веб-интерфейс, а публичный домен предоставляет пользователям удобную точку входа посредством перенаправления на активный Cloudflare Tunnel.
