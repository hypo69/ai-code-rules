# Единый стандарт разработки кроссплатформенного приложения
**Платформы:** Web, Telegram Mini App, Android Mobile, Android TV (MAG на Android TV)  
**Архитектурный паттерн:** Web-First (Single Page Application) с адаптерами сред  

---

## 1. Архитектурная концепция

Построение приложения на базе единой веб-кодовой базы (React / Vue + TypeScript) с последующей адаптацией под целевые платформы за счет изоляции логики сред и использования унифицированного API-клиента.

```
                  +----------------------------------------+
                  |         Общее веб-приложение (SPA)     |
                  +----------------------------------------+
                                       |
       +-------------------------------+-------------------------------+
       |                               |                               |
       v                               v                               v
 [ Web / TMA ]               [ Mobile (Android) ]            [ MAG на Android TV ]
(Стандартный веб)             (Capacitor Wrapper)            (Capacitor + D-pad)
       |                               |                               |
       +-------------------------------+-------------------------------+
                                       |
                                       v
                  +----------------------------------------+
                  |            API-клиент (Axios)          |
                  |     (Динамический URL и авторизация)   |
                  +----------------------------------------+
```

### Среды исполнения:
1. **Web:** Доступ через стандартный браузер персонального компьютера или мобильного устройства.
2. **Telegram Mini App (TMA):** Запуск веб-приложения внутри WebView Telegram с инициализацией Telegram SDK.
3. **Android (Mobile & MAG на Android TV):** Упаковка веб-приложения с помощью **Capacitor.js** в нативный APK-пакет.
4. **[Опция] Smart TV:
 -  через браузер:** Отображение веб-версии приложения в браузере любого совместимого телевизора.
 -  через smartTv приложение (пока только в проекте)

## 2. Стандарт сетевого взаимодействия (API)

Обеспечение прозрачного сетевого взаимодействия с локальным медиасервером. Конфигурирование API-клиента согласно установленным требованиям динамической настройки.

### 2.1 Динамическая настройка адреса сервера (Base URL)
Исключение жестко закодированных адресов. Чтение IP-адреса локального сервера из локального хранилища устройства с автоматическим переключением на дефолтный адрес конфигурации.

### 2.2 Реализация клиента (TypeScript)

```typescript
/**
 * =============================================================================
 * Process Name: Dynamic API Client Initialization
 * =============================================================================
 * Description:
 *   Configuration of Axios instance with dynamic authorization header injection.
 *   Adapts credentials based on TMA initData or standard Bearer JWT tokens.
 *
 * Examples:
 *   import apiClient from './apiClient';
 *   const data = await apiClient.get('/movies');
 *
 * File: apiClient.ts
 * Project: Our Intelligent Assistant
 * Module: Network
 * Author: hypo69
 * Copyright: © 2026 hypo69
 * =============================================================================
 */

import axios, { InternalAxiosRequestConfig } from 'axios';
import { platformService } from './platformService';
import { retrieveLaunchParams } from '@telegram-apps/sdk';

const apiClient = axios.create({
  timeout: 15000,
  headers: {
    'Content-Type': 'application/json',
  },
});

apiClient.interceptors.request.use(
  (config: InternalAxiosRequestConfig): InternalAxiosRequestConfig => {
    // Retrieval of dynamically saved server IP from local storage
    const localServerIp = localStorage.getItem('local_server_ip') ?? '';
    
    if (localServerIp) {
      config.baseURL = `http://${localServerIp}/api/v1`;
    } else {
      config.baseURL = import.meta.env.VITE_DEFAULT_API_URL ?? '';
    }

    const platform = platformService.getPlatform();
    config.headers.set('X-App-Platform', platform);

    if (platform === 'tma') {
      try {
        // Extraction of Telegram initData raw parameters
        const { initDataRaw } = retrieveLaunchParams();
        if (initDataRaw) {
          config.headers.set('Authorization', `tma ${initDataRaw}`);
        }
      } catch (e) {
        // Prevention of failure if execution is outside Telegram
      }
    } else {
      // Extraction of standard JSON Web Token
      const token = localStorage.getItem('auth_token') ?? '';
      if (token) {
        config.headers.set('Authorization', `Bearer ${token}`);
      }
    }

    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

export default apiClient;
```

## 4. [Опционально] Стандарт адаптации интерфейса под ТВ

Организация управления с пульта (D-pad) при запуске приложения на приставке MAG на Android TV или через встроенный браузер телевизора.

### 4.1 Логика определения платформы (`platformService.ts`)

```typescript
/**
 * =============================================================================
 * Process Name: Client Platform Identification
 * =============================================================================
 * Description:
 *   Detection of the current runtime environment (Web, Telegram Mini App, or Android).
 *   Execution of platform-specific application termination workflows.
 *
 * Examples:
 *   const platform = platformService.getPlatform();
 *   platformService.closeApp();
 *
 * File: platformService.ts
 * Project: Our Intelligent Assistant
 * Module: Core
 * Author: hypo69
 * Copyright: © 2026 hypo69
 * =============================================================================
 */

import { isTMA } from '@telegram-apps/sdk';
import { CapApp } from '@capacitor/app';

export type PlatformType = 'web' | 'tma' | 'android';

export const platformService = {
  /**
   * Identification of the active runtime platform.
   *
   * Args:
   *   None
   *
   * Returns:
   *   PlatformType — Identified platform string value ('web' | 'tma' | 'android').
   *
   * Examples:
   *   const platform = platformService.getPlatform();
   */
  getPlatform(): PlatformType {
    // Checking for Telegram Mini App environment
    if (isTMA()) {
      return 'tma';
    }
    // Checking for Capacitor Android environment
    if (Object.prototype.hasOwnProperty.call(window, 'Capacitor')) {
      return 'android';
    }
    return 'web';
  },

  /**
   * Execution of application termination based on runtime platform.
   *
   * Args:
   *   None
   *
   * Returns:
   *   boolean — Verification of exit command execution.
   *
   * Examples:
   *   platformService.closeApp();
   */
  closeApp(): boolean {
    const platform = this.getPlatform();
    if (platform === 'tma') {
      // Telegram WebApp termination
      if (window.Telegram?.WebApp) {
        window.Telegram.WebApp.close();
        return true;
      }
    } else if (platform === 'android') {
      // Capacitor app exit execution
      CapApp.exitApp();
      return true;
    } else {
      // Browser window closure attempt
      window.close();
      return true;
    }
    return false;
  }
};
```

### 4.2 Сопоставление кодов клавиш пульта (`keyHandler.ts`)

```typescript
/**
 * =============================================================================
 * Process Name: Television Keyboard Code Mapping
 * =============================================================================
 * Description:
 *   Normalization of hardware keycodes for Smart TV controllers (MAG, Tizen, webOS).
 *   Ensures consistent spatial navigation translation.
 *
 * Examples:
 *   const keyAction = mapTvKey(event.keyCode);
 *
 * File: keyHandler.ts
 * Project: Our Intelligent Assistant
 * Module: TV
 * Author: hypo69
 * Copyright: © 2026 hypo69
 * =============================================================================
 */

export const TV_KEYS = {
  ENTER: [13, 29443],
  BACK: [10009, 461, 8, 27],
  UP: [38, 29460],
  DOWN: [40, 29461],
  LEFT: [37, 4],
  RIGHT: [39, 5]
};

/**
 * Parsing of physical device keycode to standard string representation.
 *
 * Args:
 *   keyCode (number) — Device keycode event parameter.
 *
 * Returns:
 *   string — Normalized key event string action, or empty string if unmapped.
 *
 * Examples:
 *   const action = mapTvKey(13); // Returns 'ENTER'
 */
export const mapTvKey = (keyCode: number): string => {
  if (TV_KEYS.ENTER.includes(keyCode)) {
    return 'ENTER';
  }
  if (TV_KEYS.BACK.includes(keyCode)) {
    return 'BACK';
  }
  if (TV_KEYS.UP.includes(keyCode)) {
    return 'UP';
  }
  if (TV_KEYS.DOWN.includes(keyCode)) {
    return 'DOWN';
  }
  if (TV_KEYS.LEFT.includes(keyCode)) {
    return 'LEFT';
  }
  if (TV_KEYS.RIGHT.includes(keyCode)) {
    return 'RIGHT';
  }
  return '';
};
```

## 5. Стандарт разработки утилиты голосового ввода (Text <-> Speech <-> Text)

Для обеспечения стабильного голосового ввода на русском, английском и иврите на всех целевых устройствах утвержден **серверно-ориентированный (API-driven) стандарт**.

```
   [ КЛИЕНТ ]  -- (Запись звука через MediaRecorder) ->  [ POST /speech/transcribe ]  --> [ СЕРВЕР (STT: Whisper) ]
                                                                                                  |
   [ КЛИЕНТ ]  <- (Проигрывание через Audio API) <--  [ POST /speech/synthesize ]  <--  [ СЕРВЕР (TTS: Piper/Coqui) ]
```

### 5.1 Модуль Speech-to-Text (Распознавание речи)
* **Серверный движок:** **Faster-Whisper** (оптимизированный Whisper от OpenAI, развернутый локально). Обеспечивает мультиязычное распознавание без необходимости переключения языковых моделей.
* **Клиентская запись:** Использование стандартного браузерного интерфейса `MediaRecorder`.
* **API Контракт транскрибации:**
  * Метод: `POST /api/v1/speech/transcribe`
  * Формат запроса: `multipart/form-data` (аудиофайл в формате `.webm` или `.ogg`), `language` (`ru` | `en` | `he` | `auto`).
  * Формат ответа: `{ "success": true, "text": "Текст" }`

### 5.2 Модуль Text-to-Speech (Синтез речи)
* **Серверный движок:** **Piper** (для быстрой локальной генерации на русском и английском языках) или внешние нейросетевые API (для качественного синтеза иврита с автоматическим разбором огласовок).
* **Воспроизведение:** Получение бинарного аудиопотока и воспроизведение через HTML5 `Audio` API.
* **API Контракт синтеза:**
  * Метод: `POST /api/v1/speech/synthesize`
  * Формат запроса: JSON `{ "text": "Текст", "lang": "ru"|"en"|"he" }`
  * Формат ответа: Аудиопоток (MIME-тип `audio/mpeg` или `audio/wav`).

### 5.3 Реализация утилиты голосового ввода (TypeScript)

```typescript
/**
 * =============================================================================
 * Process Name: Voice Interaction Utility
 * =============================================================================
 * Description:
 *   Management of microphone recording (STT) and server-side speech synthesis playback (TTS).
 *   Supports Russian, English, and Hebrew localization.
 *
 * Examples:
 *   const voiceUtil = new VoiceUtility();
 *   await voiceUtil.startListening();
 *   const text = await voiceUtil.stopAndTranscribe('he');
 *   await voiceUtil.speak("שלום", 'he');
 *
 * File: voiceUtility.ts
 * Project: Our Intelligent Assistant
 * Module: Speech
 * Class: VoiceUtility
 * Author: hypo69
 * Copyright: © 2026 hypo69
 * =============================================================================
 */

import apiClient from './apiClient';

export class VoiceUtility {
  private mediaRecorder?: MediaRecorder;
  private audioChunks: Blob[] = [];

  /**
   * Initialization and activation of microphone recording session.
   *
   * Args:
   *   None
   *
   * Returns:
   *   Promise<boolean> — Verification of recording stream activation success.
   *
   * Exceptions:
   *   Error — Thrown on failure to capture user audio device.
   *
   * Examples:
   *   const voiceUtil = new VoiceUtility();
   *   await voiceUtil.startListening();
   */
  async startListening(): Promise<boolean> {
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    this.audioChunks = [];
    this.mediaRecorder = new MediaRecorder(stream, { mimeType: 'audio/webm' });

    this.mediaRecorder.ondataavailable = (event: BlobEvent) => {
      if (event.data.size > 0) {
        this.audioChunks.push(event.data);
      }
    };

    this.mediaRecorder.start();
    return true;
  }

  /**
   * Termination of recording session and transcription payload transmission.
   *
   * Args:
   *   lang (string) — Selected language code ('ru' | 'en' | 'he').
   *
   * Returns:
   *   Promise<string> — Resolved transcribed text response from server, or empty string on failure.
   *
   * Exceptions:
   *   Error — Thrown on recording state failure or networking issues.
   *
   * Examples:
   *   const text = await voiceUtil.stopAndTranscribe('ru');
   */
  async stopAndTranscribe(lang: 'ru' | 'en' | 'he'): Promise<string> {
    return new Promise((resolve, reject) => {
      if (!this.mediaRecorder) {
        return reject(new Error("Recording state has not been initialized"));
      }

      this.mediaRecorder.onstop = async () => {
        const audioBlob = new Blob(this.audioChunks, { type: 'audio/webm' });
        const formData = new FormData();
        formData.append('file', audioBlob, 'voice.webm');
        formData.append('language', lang);

        try {
          const response = await apiClient.post<{ text: string }>('/speech/transcribe', formData, {
            headers: { 'Content-Type': 'multipart/form-data' }
          });
          resolve(response.data.text ?? '');
        } catch (error) {
          reject(error);
        } finally {
          if (this.mediaRecorder?.stream) {
            this.mediaRecorder.stream.getTracks().forEach((track) => track.stop());
          }
        }
      };

      this.mediaRecorder.stop();
    });
  }

  /**
   * Execution of server-side speech synthesis streaming and local playback.
   *
   * Args:
   *   text (string) — Input text content for generation.
   *   lang (string) — Language locale selection ('ru' | 'en' | 'he').
   *
   * Returns:
   *   Promise<boolean> — Playback initiation success verification.
   *
   * Examples:
   *   await voiceUtil.speak("Hello world", 'en');
   */
  async speak(text: string, lang: 'ru' | 'en' | 'he'): Promise<boolean> {
    if (!text) {
      return false;
    }
    try {
      const response = await apiClient.post<Blob>('/speech/synthesize', { text, lang }, { responseType: 'blob' });
      const audioUrl = URL.createObjectURL(new Blob([response.data], { type: 'audio/mpeg' }));
      const audio = new Audio(audioUrl);
      await audio.play();
      return true;
    } catch (error) {
      return false;
    }
  }
}
```
